# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

### Данные

В приложении используются две основные сущности данных: товар и покупатель.

#### интерфейс товара IProduct

interface IProduct {
  id: string;
  description: string;
  image: string;
  title: string;
  category: string;
  price: number | null;
}

Назначение: Описывает структуру объекта товара, получаемого с сервера и используемого для отображения карточки в каталоге, в корзине и в заказе.

#### Интерфейс покупателя IBuyer

type TPayment = 'online' | 'cash' | '';

interface IBuyer {
  payment: TPayment;
  email: string;
  phone: string;
  address: string;
}

Назначение: Хранит данные покупателя, необходимые для оформления заказа: способ оплаты, контактные данные и адрес доставки.

### Модели данных

В архитектуре приложения используется три модели данных.
Каждая отвечает за строго определённую функциональность и не пересекается с зонами ответственности других моделей.

#### Класс MainPageCatalog

Назначение: Хранит и управляет каталогом товаров, отображаемых на главной странице приложения.

Конструктор пустой. Начальные данные модели не задаются.
constructor()

Поля класса:  
`products: IProduct[]` - Массив всех товаров, полученных с сервера
selectedProduct: IProduct | null = null - Товар, выбранный для просмотра в подробной карточке

Методы класса:  
`setProducts(products: IProduct[]): void` - Сохраняет массив товаров в модель данных.
Тип входных данных: массив объектов интерфейса IProduct.
`getProducts(): IProduct[]` - Возвращает копию массива всех товаров.
`getProductById(id: string): IProduct \| undefined` - Возвращает товар по его id.
`setSelectedProduct(id: string \| null): void` - Сохраняет товар, выбранный для подробного просмотра. Если товара с указанным id нет — сохраняет null.
`getSelectedProduct(): IProduct \| null` - Возвращает выбранный товар или null.

#### Класс Basket

Назначение: Хранит товары, которые пользователь выбрал для покупки (корзина).

Конструктор пустой, корзина всегда начинается пустой.
constructor()

Поля класса:  
`items: IProduct[]` - Список товаров, добавленных в корзину

Методы класса:  
`getItems(): IProduct[]` - Возвращает массив товаров из корзины.
`add(product: IProduct): void` - Добавляет товар в корзину.
`remove(product: IProduct): void` - Удаляет товар из корзины.
`clear(): void` - Удаляет все товары из корзины.
`getTotalPrice(): number` - Возвращает сумму стоимости всех товаров.
`getCount(): number` - Возвращает количество товаров в корзине.
`hasProduct(productId: string): boolean` - Проверяет, есть ли товар по указанному id.
`getItemIds(): string[]` - Возвращает массив id товаров, используется для оформления заказа.

#### Класс BuyerData

Назначение: Хранит данные покупателя и отвечает за их валидацию.

Конструктор без параметров. Все поля инициализируются пустыми строками.
constructor()

Поля класса:  
`payment: IBuyer['payment']` -	Способ оплаты 
`address: string` - Адрес доставки
`email: string` - Электронная почта
`phone: string` - Телефон покупателя

Методы класса:  
`setData(data: Partial<IBuyer>): void` - Сохраняет любые поля покупателя.
Позволяет обновлять только отдельные поля, не перезаписывая остальные.
`getData(): IBuyer` - Возвращает полный набор данных покупателя.
`clear(): void` - Очищает все данные покупателя.
`validate(): Partial<Record<keyof IBuyer, string>>` - Валидация данных покупателя.
Возвращает объект ошибок — ключи только с некорректными полями.

Пример:
{
  payment: 'Не выбран способ оплаты',
  email: 'Укажите email'
}

Если ошибок нет — возвращается пустой объект.

### Слой коммуникации

#### Класс CommunicationAPI

Назначение: Абстракция над серверными запросами.
Отвечает только за обмен данными с backend API.

Конструктор
constructor(api: IApi)

Параметры:
`api: IApi` - Экземпляр низкоуровневого клиента API, предоставленного в стартовом наборе

Методы класса:  
`getProducts(): Promise<IApiProductList>` - Запрашивает каталог товаров с сервера (GET /product/). Возвращает объект с массивом товаров.
`postOrder(data: IOrderRequest): Promise<IOrderResponse>` - Отправляет заказ на сервер (POST /order/).
Принимает данные покупателя и список id товаров.
Возвращает объект с результатом оформления заказа.

### Слой представления (View)

Слой представления отвечает за отображение данных, работу с DOM-разметкой и реакцию на действия пользователя.  
Все компоненты View:

- наследуются от базового `Component<T>` из `src/components/base/Component.ts`;
- не содержат бизнес-логики (логика только в презентере);
- не хранят данные (только получают их извне);
- генерируют события через `EventEmitter`;
- работают с шаблонами (`<template>`) в `index.html`.

Компоненты View на основе шаблонов:

- `#card-catalog` — карточка товара в каталоге
- `#card-preview` — карточка товара в модальном окне (просмотр)
- `#card-basket` — карточка товара в корзине
- `#order` — форма выбора оплаты и адреса
- `#contacts` — форма email/телефона
- `#success` — сообщение об успешной оплате
- `#modal-container` — модальное окно
- `<main class="gallery">` — контейнер каталога
- `.header__basket` — кнопка корзины и счётчик

#### Базовый класс Card
Родитель для всех карточек товара независимо от их назначения.

Конструктор:  
`constructor(template: HTMLTemplateElement)`

Поля класса:  
`_element: HTMLElement` - корневой элемент карточки
`_title: HTMLElement` - название товара
`_price: HTMLElement` - цена
`_image: HTMLImageElement` - изображение товара
`_category: HTMLElement` - категория товара
`_button: HTMLButtonElement` - управляющая кнопка

Методы:  
`getElement(): HTMLElement` - Возвращает корневой DOM-элемент карточки, созданный из переданного шаблона. Используется презентером и другими компонентами для вставки карточки в DOM.
`setTitle(title: string): void` - Устанавливает название товара внутри карточки. Использует элемент `.card__title` внутри шаблона. Используeтся для установки названия в каталоге, в карточке просмотра, в карточке корзины.
`setPrice(price: number | null): void` - Устанавливает отображаемую цену товара.
Поведение: если цена число, отображается как <цена>. Если цена null, карточка отображает состояние «Бесценно». Нужно для каталога, просмотра товара, корзины, отображения заблокированной кнопки, если товар недоступен.
`setImage(src: string, alt?: string): void` - Устанавливает изображение товара. Используется во всех видах карточек, так как в каталоге и превью требуется отображать фото товара.
`setCategory(category: string): void` - Устанавливает категорию товара и применяет визуальный модификатор фона для категории. Использует объект categoryMap из src/utils/constants.ts.
`setButtonText(text: string): void` - Меняет текст кнопки карточки.
Используется для «Купить», «Удалить из корзины», «Недоступно». 
Позволяет презентеру динамически переключать состояние карточки.
`setButtonDisabled(disabled: boolean): void` - Активирует или деактивирует кнопку карточки.
Используется:
- если price === null → кнопка отключается (disabled)
- если товар уже в корзине → кнопка активна/меняет текст
- в превью карточке при просмотре

#### Класс PreviewCard (наследник Card)
Назначение: подробная карточка товара в модальном окне. Использует шаблон `#card-preview` из `index.html`.

Дополнительные поля класса:  
`_description: HTMLElement` - Описание товара. Используется для вывода полной информации о товаре, которая отсутствует в карточке каталога.

Методы:  
`setDescription(text: string): void` - Устанавливает подробное текстовое описание товара.

Генерируемые события:  
`preview:add-to-basket` - Вызывается, когда пользователь нажимает кнопку «Купить» внутри превью.  
Презентер должен:
- добавить товар в корзину,
- закрыть модальное окно,
- обновить вывод списка и счётчика корзины.
`preview:remove-from-basket` - Вызывается при нажатии кнопки «Удалить из корзины» внутри превью.  
Презентер должен:
- удалить товар из корзины,
- закрыть модальное окно,
- обновить корзину и каталожную карточку (кнопка должна поменяться).

Особенности PreviewCard:  
- отображается только в модальном окне;
- показывает полное описание товара, отсутствующее в карточке каталога;
- кнопка может находиться в трёх состояниях:
  - «Купить»
  - «Удалить из корзины»
  - «Недоступно» (если `price === null`);
- не содержит бизнес-логики — только презентация и генерация событий.

`setAddressError(message?: string): void` - Отображает ошибку, связанную с полем адреса.

Генерируемое событие:  
`order:submit-step1` - уведомляет презентер, что пользователь хочет перейти ко второму шагу.

#### Класс SuccessView

### События слоя View

В представлении генерируются следующие события:

`card:select` - Выбрать карточку товара.
`preview:add-to-basket` - Добавить товар в корзину в модальном окне Превью.
`preview:remove-from-basket` - Удалить товар из корзины в модальном окне Превью.
`basket:remove-item` - Удалить товар из корзины в модальном окне Корзины.
`basket:checkout` - переход к первому шагу оформления Заказа по клику на кнопку "Оформить" в модальном окне Корзины.
`order:submit-step1` - переход ко второму шагу оформления Заказа по кнопке "Далее".
`order:submit-step2` - завершение процесса оформления Заказа по кнопке "Оплатить".
`success:close` - закрытие модального окна с сообщением об успешной оплате по кнопке «За новыми покупками!».

Все события обрабатываются только презентером.  
Компоненты View не меняют данные и не вызывают методы моделей — они лишь уведомляют о действиях пользователя.
